# Plan - 스킬 오케스트레이터

프로젝트 목표를 분석하여 최적의 스킬 조합을 제안하고, 스킬 생태계를 지속적으로 발전시킵니다.

## 실행 방법

- `/plan "프로젝트 설명"` 또는 "플랜 짜줘"

## 작동 방식

```
[입력] 프로젝트 목표/설명
         │
         ▼
[1단계] 프로젝트 분석
         │ - 프로젝트 타입 식별
         │ - 필요 기능 추출
         │ - 기술 스택 확인
         ▼
[2단계] 스킬 매칭
         │ - 기존 스킬 중 적합한 것 선택
         │ - 실행 순서 결정
         │ - 스킬 간 의존성 파악
         ▼
[3단계] 갭 분석
         │ - 부족한 스킬 식별
         │ - 기존 스킬 개선점 파악
         │ - 새 스킬 제안
         ▼
[출력] 실행 계획 + 스킬 제안
```

## 프로젝트 타입 분류

| 타입 | 키워드 | 핵심 스킬 |
|------|--------|----------|
| **SaaS** | 대시보드, 관리자, 구독 | next-page, react-hook, api-route, ts-form |
| **이커머스** | 쇼핑몰, 상품, 결제 | + tw-component, gen-ai |
| **AI 앱** | 생성, 챗봇, 이미지 | api-route, react-hook, gen-ai |
| **랜딩** | 소개, 마케팅, CTA | next-page, tw-component, ftw |
| **블로그/CMS** | 글, 콘텐츠, MDX | next-page, api-route |
| **모바일웹** | 앱, PWA, 반응형 | tw-component, react-hook |

## 스킬 제안 기준

### 새 스킬 제안 트리거

| 상황 | 제안할 스킬 |
|------|------------|
| 로그인/회원가입 언급 | `/auth` |
| DB/Prisma/Supabase 언급 | `/db` |
| 테스트/TDD 언급 | `/test` |
| 배포/CI/CD 언급 | `/deploy` |
| SEO/메타태그 언급 | `/seo` |
| 국제화/다국어 언급 | `/i18n` |
| 결제/구독 언급 | `/payment` |
| 이메일/알림 언급 | `/notify` |

### 기존 스킬 업데이트 트리거

| 상황 | 업데이트 제안 |
|------|--------------|
| 새 라이브러리 사용 | 해당 패턴 추가 |
| 반복되는 패턴 발견 | 스킬에 패턴화 |
| 에러 자주 발생 | 가이드라인 추가 |
| 새 Next.js 버전 | 최신 문법 반영 |
